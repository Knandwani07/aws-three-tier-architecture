# 📚 Documentation

This folder contains the supporting documentation for the **AWS Three-Tier Architecture** project. Each document explains a specific phase of the deployment process, from infrastructure provisioning and execution to verification, command references, and resource cleanup.

Together, these documents provide a complete guide for deploying, understanding, validating, and managing the three-tier architecture using AWS best practices.

---

## 📂 Contents

| File | Description |
|------|-------------|
| `deployment-guide.md` | Step-by-step guide for deploying the complete AWS Three-Tier Architecture. |
| `execution-workflow.md` | Explains how requests flow through the architecture and how AWS services interact during application execution. |
| `command-reference.md` | Lists the Linux, AWS CLI, and MySQL commands used throughout the project with a brief explanation of each command. |
| `cleanup-guide.md` | Provides the recommended order for deleting AWS resources to prevent unnecessary charges after completing the project. |

---

## 🎯 Purpose

The documentation in this folder helps users:

- Deploy the complete AWS Three-Tier Architecture.
- Understand the architecture's request flow and component interactions.
- Reference commonly used Linux, AWS CLI, and MySQL commands.
- Safely clean up AWS resources after project completion.
- Learn AWS networking, security, and infrastructure deployment best practices.

These documents complement the architecture diagrams and screenshots, providing a comprehensive walkthrough of the project's implementation and operation.
